# ガードレール（禁止事項）

## セキュリティ

1. **絶対にクライアントに`service_role_key`を露出しない**
2. **個人情報をログに記録しない**
3. **SQLインジェクション対策を必須とする**
4. **RLSを必ず有効化する**

## データ保護

1. **他部署のデータへの不正アクセスを防ぐ**
2. **削除済みデータの復元を禁止**
3. **アテステーション記録の改ざんを防ぐ**

## AI利用

1. **不適切なコンテンツ生成を防ぐ**
2. **偏った情報や誤情報の配信を防ぐ**
3. **AI生成コンテンツは必ず人間がレビュー**

## コンプライアンス

1. **アテステーションの強制は法務確認後のみ**
2. **監査ログの無断削除・編集を禁止**
3. **個人情報の取り扱いは各国法令に準拠**

## パフォーマンス

1. **一度に1000件以上のデータ取得を禁止**
2. **無限ループや再帰処理を防ぐ**
3. **キャッシュ戦略を必須とする**

## ユーザー体験

1. **誤解を招くUI/UXを作らない**
2. **必須項目を明確に表示**
3. **エラーメッセージは具体的に**
